68  /*
69  Event handles:
70  You need some callbacks to get your intended code executed when the related event occurred.
71  for callback references input supports two kinds of callbacks:
72  1.: The (non object orientated) callback function pointer:
73  void (*callbackFunction)();
74  You can give input a pointer to your void-terminated function to call on event.
75  The functions doesn't get any arguments when called.
76  (Hint: You can get additional information by using getEventCode() or gbutton, gchange)
77  2.: The (object orientated) callback objects:
78  Callable* callbackObject;
79  You can give input a pointer to a object of a class
80  derived from Callable (defined in this library)
81  The (virtual) method operator() will be called when the event occurs.
82  operator() takes no arguments and returns in void such as in case (1)
83 
84  Notice: In C++ it is not possible to use a simple (C-like) callback function pointer in order to
85  point to a member function of some object. If you use class members you should use (2).
86  In other cases it might be better to use C-like function pointers to not pollute µC memory.
87  If you (only) want to point to static (class) methods there won't be any problem with using (1).
88 
89  You can always use only one of both kinds of callbacks for each single button event
90  but you can change each callback reference including its type of referring anytime you want.
91  Different buttons may have different kinds of callbacks at the same time.
92 
93  In order to this behavior callbacks are nested in a union to save bytes of RAM
94  */
